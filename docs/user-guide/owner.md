# คู่มือใช้งาน — Owner / ผู้บริหารระดับสูง

> สำหรับ: เจ้าของบริษัท · CEO · กรรมการผู้จัดการ · ผู้ที่มีตำแหน่งในระบบเป็น `super_admin` หรือ `org_admin`
> Last updated: 2026-05-20

---

## 📌 1. ภาพรวม

คุณคือคนที่ "เห็นทุกอย่าง · ตัดสินใจทุกเรื่อง" ในระบบ Pooilgroup ERP

ในมุมระบบ คุณมีสิทธิ์สูงสุด:

- เห็นยอดขายของ **ทุกสาขา ทุกบริษัทในเครือ** (ไม่จำกัดเขต)
- เชิญ / ลบพนักงาน · กำหนดบทบาท
- ตั้งค่าระบบทั้งหมด · เปิด-ปิด module ต่อบริษัท
- **ปลดล็อกรายงานที่อนุมัติไปแล้ว** (สิทธิ์เฉพาะ `super_admin` เท่านั้น — `org_admin` ทำไม่ได้)
- ดู audit log ทุกการเคลื่อนไหวในระบบ

> 💡 ความต่าง `super_admin` vs `org_admin`:
> - `super_admin` = เจ้าของจริง · ปลดล็อกรายงานที่อนุมัติแล้วได้
> - `org_admin` = ผู้ดูแลระดับองค์กร · ตั้งค่าได้เกือบทุกอย่าง แต่ปลดล็อกรายงานไม่ได้

## 🚦 2. เข้าระบบ

### ครั้งแรก (ถ้าเพิ่งเริ่ม)

1. เปิด [pooilgroup.vercel.app](https://pooilgroup.vercel.app)
2. ถ้ายังไม่มีบัญชี → คนแรกที่สมัครคือ Owner อัตโนมัติ (ผ่าน invite token ที่ทีม setup ส่งให้)
3. ตั้งรหัสผ่าน → เข้าระบบ

### ครั้งต่อไป

1. เปิด [pooilgroup.vercel.app/login](https://pooilgroup.vercel.app/login)
2. กรอกอีเมล + รหัสผ่าน → กด "เข้าสู่ระบบ"
3. ระบบพาเข้า `/home` อัตโนมัติ

[screenshot: หน้า login จะใส่ภายหลัง]

## ✅ 3. งานประจำวัน (Daily Tasks)

### 3.1 ตรวจยอดขายเมื่อวานครบทุกสาขา

**ทำอะไร:** ดูว่าทุกสาขาส่งรายงานครบหรือยัง · ใครยังไม่ส่ง

1. เข้า `/home` หรือ `/cashhub/dashboard`
2. ดู section "ภาพรวมเดือนนี้" — เห็นยอดรวม + จำนวนสาขาที่ส่งครบ
3. ดู section "ยังไม่ส่งรายงาน" — ถ้ามีสาขาค้างหลายวัน → กดเข้าไปดูรายละเอียด

**ผลลัพธ์ที่คาดหวัง:** เห็นยอดรวมเดือนนี้ vs target · เห็น forecast สิ้นเดือน · เห็นสาขาที่ผิดปกติ

[screenshot: หน้า dashboard จะใส่ภายหลัง]

### 3.2 อนุมัติรายงาน (ถ้าไม่ delegated ให้ Branch Manager)

**ทำอะไร:** ตรวจรายงานยอดประจำวันก่อนล็อกเข้าระบบบัญชี

1. เข้า `/cashhub/reports`
2. กรองด้วย chip "รออนุมัติ" (status = submitted)
3. กดเข้าไปดูรายงานทีละใบ — ตรวจยอดเงิน · ตรวจ reconcile (ยอด vs เงินจริง)
4. ถ้าตรง → กด **อนุมัติ** · ถ้าไม่ตรง → กด **ไม่อนุมัติ** + ใส่เหตุผล
5. ใช้ **Quick Approve Bar** ถ้าจะอนุมัติทีละหลายใบพร้อมกัน

> 💡 ถ้าอยากให้ Branch Manager อนุมัติแทน → assign ผู้จัดการสาขาเข้า branch (ดูข้อ 4.2)

### 3.3 ตรวจเงินขาด / เงินเกิน

**ทำอะไร:** ดูสาขาที่มีเงินขาด — ใครรับผิดชอบ · เหตุผล

1. เข้า `/cashhub/shortages`
2. กรองตามวันที่ / สาขา / คนที่รับผิดชอบ
3. ดูรายละเอียดเงินขาด — ระบบบันทึกว่าใครรับผิดชอบ (จาก shortage modal)

## 📅 4. งานประจำสัปดาห์ / เดือน

### 4.1 ตรวจ Health Score ของแต่ละสาขา

**เมื่อไหร่:** ทุกวันจันทร์ตอนเช้า · หรือสิ้นเดือน

1. เข้า `/cashhub/leaderboard` — เห็นสาขาเรียงจากดีที่สุด → แย่ที่สุด
2. ดู Health Score (A–F) ของแต่ละสาขา
3. กรองด้วย business type (ปั๊ม · เช่า · ฯลฯ) ถ้าต้องการ
4. กดเข้าสาขาที่ได้คะแนน D หรือ F → ดูเหตุผล (ส่งช้า · ส่งไม่ครบ · เงินขาดบ่อย)

**Action:** ถ้าสาขาคะแนนแย่ — โทรหาผู้จัดการสาขา · หรือไปดูที่ `/cashhub/branches/[id]`

### 4.2 เพิ่ม / เชิญพนักงานใหม่

**ทำอะไร:** เปิดสิทธิ์ให้พนักงานใหม่เข้าระบบ

1. เข้า `/users`
2. กดปุ่ม **เชิญผู้ใช้** (มุมบนขวา)
3. กรอก:
   - ชื่อ-นามสกุล
   - อีเมล (จะส่งลิงก์เชิญไปที่นี่)
   - บทบาท (super_admin / org_admin / admin / area_manager / branch_manager / staff / driver)
   - สาขาที่ assigned (ถ้าเป็น staff / branch_manager)
4. กดเชิญ → ระบบสร้างลิงก์ → copy ไปส่งให้พนักงานทาง LINE
5. พนักงานกดลิงก์ → ตั้งรหัสผ่าน → เข้าระบบได้

> 💡 บทบาทแต่ละแบบทำอะไรได้:
> - `staff` = พนักงานหน้าสาขา (กรอก · ดู ยอดของตัวเอง)
> - `branch_manager` = ผจก.สาขาเดียว (อนุมัติของสาขาตัวเอง)
> - `area_manager` = ดูแลหลายสาขา (กรอกแทนสาขาในเขตได้)
> - `admin` = ออฟฟิศกลาง (อนุมัติทุกสาขาแต่ปลดล็อกไม่ได้)
> - `org_admin` = ผู้ดูแลระดับองค์กร
> - `super_admin` = เจ้าของจริง (ปลดล็อกได้)
> - `driver` = พนักงานขับรถ (ใช้ใน FuelOS · ยังไม่เปิดใช้)

[screenshot: หน้า users จะใส่ภายหลัง]

### 4.3 ตรวจ audit log สิ้นเดือน

**ทำอะไร:** ดูว่าใครทำอะไรในระบบช่วงเดือนที่ผ่านมา

1. เข้า `/audit`
2. ดูรายการเรียงตามเวลา (ใหม่สุดอยู่บน)
3. กรอง (ถ้ามีปัญหา) — เช่น "ดูเฉพาะการอนุมัติ" "ดูเฉพาะคนนี้"
4. มองหาสิ่งผิดปกติ — เช่น ปลดล็อกรายงานเก่า · ลบพนักงาน · เปลี่ยน setting

### 4.4 ตรวจเอกสารใกล้หมดอายุ

**ทำอะไร:** กันลืมต่ออายุใบอนุญาต · ใบขับขี่ · ทะเบียนรถ

1. เข้า `/docuflow/expiry`
2. ดู 4 กลุ่ม: หมดอายุแล้ว · วิกฤต ≤7 วัน · ด่วน ≤30 วัน · เฝ้าระวัง ≤90 วัน
3. กดเข้าเอกสาร → ดู renewal workflow · มอบหมายผู้รับผิดชอบ

[screenshot: หน้า docuflow/expiry จะใส่ภายหลัง]

### 4.5 รายงานรายเดือน (PDF)

**ทำอะไร:** ดาวน์โหลด PDF สรุปยอดเดือนที่ผ่านมา

1. เข้า `/cashhub/monthly-report`
2. เลือกเดือน → กดปุ่ม "พิมพ์" หรือ "ดาวน์โหลด"
3. ระบบจะ generate PDF อัตโนมัติทุกวันที่ 1 ของเดือน (ถ้าตั้ง cron แล้ว)

## 🔧 5. การตั้งค่าระบบ (Settings)

### 5.1 ข้อมูลองค์กร

- เข้า `/settings`
- ดูชื่อบริษัท · slug · logo · fiscal year
- แก้ไขได้ตามต้องการ (ยกเว้น slug ที่ล็อกหลังสร้าง)

### 5.2 เปิด-ปิด module

- เข้า `/settings` → section "Module Toggle"
- เปิด / ปิด CashHub / FuelOS / DocuFlow ต่อบริษัท
- ถ้าปิด module → พนักงานจะไม่เห็นเมนูนั้น

### 5.3 แจ้งเตือน (Notifications)

- เข้า `/settings/notifications`
- ตั้งค่าเวลา cron แจ้งเตือน (Telegram / LINE / email)
- *Note:* Telegram Bot ยังอยู่ระหว่างพัฒนา · ตอนนี้ตั้งไว้แต่ยังไม่ส่งจริง

### 5.4 ความปลอดภัย

- เข้า `/settings/security`
- ตั้งค่า idle timeout · failed login lock
- *Note:* บางส่วนยังอยู่ระหว่างพัฒนา (ดู STATUS.md)

### 5.5 Backup

- เข้า `/settings/backup`
- ดาวน์โหลด backup เป็น CSV
- ตั้ง schedule auto backup รายวัน (ทำผ่าน Supabase Dashboard ตรง ๆ — ยังไม่มี UI ในแอป)

## 🛠️ 6. สิทธิ์พิเศษ — ปลดล็อกรายงาน (Unlock)

> เฉพาะ `super_admin` เท่านั้น — `org_admin` ทำไม่ได้

**กรณีที่ต้องปลดล็อก:**

- รายงานอนุมัติไปแล้ว · แต่พบข้อมูลผิด ต้องแก้
- พนักงานกรอกผิดวัน

**วิธีทำ:**

1. เข้า `/cashhub/reports/[id]` (หน้ารายละเอียดรายงานนั้น)
2. กดปุ่ม **"ปลดล็อก"** (มุมขวาบน)
3. ระบบจะเปลี่ยน status จาก approved → draft
4. พนักงานสาขาแก้ไขแล้วส่งใหม่
5. อนุมัติอีกครั้ง

> ⚠️ **ทุกการปลดล็อกถูกบันทึก audit log** · ตอบ auditor / กรมสรรพากรได้

## ❓ 7. ปัญหาที่เจอบ่อย + วิธีแก้

**Q: พนักงานกรอกรายงานไม่ได้ บอกว่า "ไม่มีสิทธิ์"**
A: เช็คว่า assign สาขาให้พนักงานคนนั้นยัง · เข้า `/users` → คลิกพนักงาน → ดู "สาขาที่ assigned" · ถ้าว่างให้ assign เข้า

**Q: รายงานอนุมัติไปแล้ว ต้องแก้ — ทำยังไง**
A: เฉพาะ `super_admin` ปลดล็อกได้ (ดูข้อ 6) · `org_admin` ต้องบอก `super_admin` ทำให้

**Q: Health Score ของสาขา A ต่ำกว่าที่ควร**
A: เข้า `/cashhub/branches/[id]` ดู breakdown — ระบบบอกว่าหักคะแนนจาก ส่งช้า / ไม่ส่ง / เงินขาด / streak ขาด

**Q: ตั้ง deadline ส่งรายงานของแต่ละสาขายังไง**
A: ตอนนี้ deadline ตั้งใน branch (default 21:00) · ยังไม่มี UI แก้ง่าย ๆ · ต้องไปแก้ผ่านหน้า branch edit (อยู่ระหว่างพัฒนา)

**Q: ลบบัญชีพนักงานที่ลาออก**
A: ตอนนี้ deactivate ยังไม่มี UI · ทีม IT ต้อง update ตรงใน database (อยู่ระหว่างพัฒนา — Phase C2)

**Q: เอกสารใน DocuFlow เข้าไม่ได้ บอก 403**
A: บางเอกสารตั้ง access level สูง — ดูว่าตำแหน่งคุณตรงกับเอกสารหรือไม่ · ถ้าเป็น super_admin ควรเข้าได้ทุกอัน

**Q: Dashboard ช้า · โหลดนาน**
A: ลอง refresh · ถ้ายังช้าให้แจ้ง IT — อาจมีปัญหา query ในช่วง peak

**Q: ใช้ในมือถือได้ไหม**
A: ได้ · responsive ดี · แต่หน้าจอใหญ่ (laptop / desktop) จะเห็นข้อมูลครบกว่า

**Q: มีคน login เครื่องคนอื่น — ดูได้ไหมว่าใคร**
A: ตอนนี้ session list ยังอยู่ระหว่างพัฒนา (Phase C1) · ดู audit log เห็นว่ามีใคร LOGIN เมื่อไหร่

**Q: เผลอลบสาขา · เผลอเปลี่ยน setting**
A: เข้า `/audit` ค้นหา event นั้น · ทีม IT restore จาก backup ได้

## 💡 10. Tips สำหรับ Owner

1. **เช็คสั้นทุกเช้า** — เปิด `/home` 30 วินาที ก็พอ · เห็นยอดเมื่อวาน + สาขาที่ค้าง
2. **ส่งต่อหน้าที่ approve** — ให้ Branch Manager ดูสาขาตัวเอง · คุณดูเฉพาะเคสที่ flag
3. **ดู Leaderboard ทุกวันจันทร์** — สาขาที่คะแนนตก 2 สัปดาห์ติด = ต้องคุย
4. **ตรวจ audit log สิ้นเดือน** — ดูว่าใครทำผิดปกติ · ก่อนสรุปเงินเข้าระบบบัญชี
5. **ใช้ Compare มอง trend** — เปรียบเทียบเดือน vs เดือน เห็นว่าธุรกิจไหนขึ้น/ลง
6. **อย่าให้ทุกคนเป็น super_admin** — เกินจำเป็น เสี่ยงอันตราย · ใช้ `admin` หรือ `org_admin` ก็พอ
7. **ตั้ง logo + ชื่อบริษัทตั้งแต่วันแรก** — มันจะปรากฏใน PDF report ที่ส่งให้ลูกค้า
