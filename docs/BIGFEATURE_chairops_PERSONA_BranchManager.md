## ระบบใหม่จะเปลี่ยนอะไรให้หนู (Wave-by-Wave)

- **Wave 0 (XLSX upload + diff preview)** — ลากไฟล์ทิ้งครั้งเดียว · ระบบ diff ให้ว่าวันไหนใหม่ วันไหนซ้ำ · หนูประหยัด **1.5 ชม./วัน** (ที่เคย copy แต่ละ tab) · ดีมากกกก
- **Wave 1 (LINE OA + Dashboard 30×4 grid)** — หน้าเดียวเห็น 30 สาขาวันนี้ใครส่ง ✓ ไม่ส่ง ✗ รอ ⏳ · ไม่ต้อง scroll LINE · กดปุ่ม "เตือน" ระบบส่ง LINE OA แทนหนู · ประหยัด **2 ชม./วัน** + ลดงานทวงซ้ำ
- **Wave 2 (Gmail AI bill parser + accounting export)** — บิลห้างเข้ามา · AI อ่านยอด · หนูแค่กดอนุมัติ · ปลายเดือนส่งบัญชี import เข้า BC ได้เลย · ประหยัด **8 ชม./เดือน**
- **Wave 3 (audit + leaderboard + EOD digest)** — มี audit page เปิดดู log ได้ · ทำ report ส่งพี่ตอนเย็นไม่ต้องนั่งสรุปเอง · ระบบสรุปให้ ประหยัด **30 นาที/วัน**

## ที่หนูกลัวว่าจะพังตอนระบบ live

1. **แม่บ้านไม่ยอมเลิก LINE group** — เขาเคยชิน · ถ้าระบบรองรับไม่ครอบคลุม เขาส่ง LINE group เหมือนเดิม · หนูจะ "หน้าทั้งสองที่"
2. **อินเทอร์เน็ตห้างขาด** — บางห้างชั้น 4 WiFi ไม่เข้า · LIFF Mini App เปิดไม่ติด · แม่บ้านบอก "ระบบเสีย" แล้วเลิกใช้
3. **AI อ่านบิลผิด** — ถ้ายอดเพี้ยน 50 บาทแล้วหนูไม่จับได้ · บัญชี import ผิด · ตอนผู้สอบมาตรวจอาจมีปัญหา
4. **Drift engine แจ้ง false alarm** — ถ้า alert "ขาดเงิน" บ่อยเกินไปทั้งที่ของแม่บ้านยังไม่มาเก็บ หนูจะเชื่อ alert น้อยลง (boy-cry-wolf)
5. **ระบบช้า/ค้าง ตอนใช้พร้อมกันหลายคน** — 30 แม่บ้านส่งพร้อมกันตอน 17:00 · ถ้าระบบช้า แม่บ้านจะกดซ้ำเป็นรอบ · ซ้ำใน DB

## สิ่งที่ spec ยัง miss (เจอทุกวันแต่ไม่อยู่ใน plan)

1. **ถุงมือ/น้ำยาทำความสะอาด ของแม่บ้าน** — หนูต้องเบิกให้ทุก 2 อาทิตย์ · ไม่มี flow ในระบบ (มีแค่ spare parts สำหรับเก้าอี้)
2. **ไฟดับ/WiFi ดับที่ห้าง** — แม่บ้านส่งไม่ได้ · ระบบควรมี "offline outbox" + manual entry override
3. **แม่บ้านลาป่วย/หยุดวันสำคัญ** — ใครเก็บแทน? · ระบบยังไม่มี substitute roster
4. **เก็บซ้อนวัน (double-collection)** — บางทีแม่บ้านมาเช้าแล้วบ่ายมาอีก · ระบบนับเป็น 1 record หรือ 2?
5. **วันหยุดยาว (สงกรานต์/ปีใหม่)** — ห้างปิด 3 วัน · ระบบควร skip drift alert วันที่ห้างปิด ไม่ใช่ alert ทุกวัน
6. **บิลห้างที่มี credit note** — ห้างคืนเงิน/ลดยอด · บัญชีต้องรู้ · ระบบ vendor-bill ตอนนี้รองรับมั้ย?
7. **คนใหม่ (พนักงานทดลอง 3 เดือน)** — ระบบสมัครยังไง · ใครอนุมัติ · ขอ access ได้แค่บางสาขามั้ย?

## คำตอบคำถามจากพี่ (CEO)

- **30×4 grid ตอนเช้าหนูจะเปิดมั้ย?** เปิด **แต่** หนูยังเปิด LINE ก่อนอยู่ 1-2 อาทิตย์เพราะติดนิสัย · ขอ shortcut บน home screen + push noti ตอน 08:00 ให้เปิด dashboard ก่อน
- **เชื่อ AI อ่านบิลมั้ย?** เชื่อ 70% · ที่เหลือหนูจะ double-check ทุกใบจนกว่า AI จะแม่น 95%+ ติดต่อกัน 2 เดือน · อย่าให้ระบบ auto-commit เด็ดขาด
- **เข้าใจ soft/hard close มั้ย?** ไม่เข้าใจ · มันคือของพี่ · หนูต้องการ tooltip + video สอน 5 นาที + "ถ้ากดผิดยกเลิกได้มั้ย"
- **อยากเพิ่มอะไร?** (a) ปุ่ม "ส่ง LINE หาแม่บ้านทุกคนที่ยังไม่ส่ง" 1 ปุ่ม · (b) Export Excel จาก dashboard (หนูยังต้องส่งให้บัญชีบางครั้ง) · (c) "หมายเหตุ" field ในแต่ละ row เผื่อจดอะไรเป็นการส่วนตัว

## Adoption test — อะไรจะทำให้หนูทิ้งระบบใหม่หลัง Day 3

1. ถ้าวันแรก dashboard โหลดเกิน 5 วินาที → หนูจะเปิด LINE แทน
2. ถ้า alert "เงินขาด" เด้งผิด 2 ครั้งใน 3 วัน → หนูจะไม่เชื่อระบบอีกเลย
3. ถ้าแม่บ้าน 5 คนใช้ LIFF ไม่เป็น + หนูต้องสอนทีละคน → หนูจะบอกพวกเขากลับไป LINE group
4. ถ้า XLSX upload error แต่ error message เป็น "Validation failed at row 17" (ภาษา dev) → หนูจะร้องไห้
5. ถ้าระบบไม่รับ "หมายเหตุ" field → หนูจะมี notebook แยก กลายเป็น 2 source of truth
